AHP 2.0 EXTREME HANDBOOK

Technical Architecture & Engineering Blueprint

01 Project Overview

AHP 2.0 is an Enterprise-Grade Healthcare Orchestrator. It provides sub-10 second AI summarization of clinical records with absolute PII privacy.

Identity

Scalable, AI-Native, Health platform for clinicians and patients.

Mission

Redefining clinical efficiency through automated medical intelligence.

02 High-Level System Architecture

                    API --> DB[(PostgreSQL)]

03 Frontend Architecture

Dual-stack architecture using Expo (Mobile) and Vite (Web). Atomic Design patterns ensure consistency across all screens.

Mobile (Expo)

Secure Store, Native Hardware Access, React Native.

Web (Vite)

Tailwind CSS, Dashboard Analytics, Real-time status updates.

04 Backend Architecture

Built with FastAPI for non-blocking I/O. Uses a Service-Oriented (SOA) pattern with dependency injection for clean, testable logic.

Chapter 04b

AI Engine Deep-Dive (The "Engen")

The AHP 2.0 Engine is an Asynchronous Priority-Based Hub. It orchestrates multiple LLM providers to ensure sub-10s latency even during provider outages.

Orchestration Logic

                graph TD
                    IN[Input: Medical Record] --> OPT[Image Optimizer]
                    OPT --> UNIFIED[Unified AI Engine]
                    UNIFIED --> Q_GROQ{Try Groq?}
                    Q_GROQ -->|Fast Success| EXT[Entity Extraction]
                    Q_GROQ -->|Rate Limit| Q_GEMINI{Try Gemini?}
                    Q_GEMINI -->|Success| EXT
                    EXT --> PYD[Pydantic Validation]
                    PYD --> PARALLEL[Parallel Summary Generation]

The "Neural Pipeline" Sequence

                sequenceDiagram
                    participant P as Pipeline
                    participant G as LLM Cluster
                    participant E as Entity Extractor
                    participant S as Summary Service

                    P->>G: High-Accuracy OCR
                    G-->>P: Raw Text
                    P->>E: JSON Schema Extraction
                    E->>G: Prompt: "Extract Medical Entities"
                    G-->>E: Structured JSON Object
                    Note over E: Validating via MedicalEntities Model
                    par Async Summarization
                        E->>S: Doctor's Clinical Summary
                    and
                        E->>S: Patient's Warm Explanation
                    end
                    S-->>P: Final Integrated Report
Multi-LLM Strategy

Uses Groq (Llama 3.2/3.3) for speed and Gemini 1.5 Flash for vision accuracy. Automated fallback ensures zero downtime.

Validation Layer

Pydantic V2 strictly enforces medical data integrity, discarding invalid AI hallucinations before they reach the DB.

05 API Architecture

RESTful JSON APIs with WebSocket support for real-time states. Versioned paths and Pydantic validation on every route.

06 Auth & Identity Mesh

Double-token JWT system (Access + Refresh). Role-Based Access Control (RBAC) enforced via decorators. Secure Bcrypt hashing with SHA-256 pre-hash.

07 Security Shields

Layers: SSL 1.3, Nginx WAF, Rate Limiting, JWT Scopes, and AES-256 Field Encryption for PII.

08 Database Mesh

PostgreSQL 16 for persistent clinical data. Redis 7.0 for session and queue management. Alembic for migrations.

09 Infrastructure Topology

Fully containerized via Docker. Self-healing restart policies. Load balanced via Nginx ingress.

11 Forensic Monitoring

Structured JSON logging, Correlation IDs for distributed tracing, and specialized audit_logs for every clinical access.

12 Cyber Resilience

Automated system repairs. RecoveryController watchdog. Hot-standby database failover. Chaos Monkey testing.

13 Threat Matrix

Mitigations against SQL Injection (ORM), XSS (Sanitization), DDoS (Nginx Rate Limits), and Brute Force (Account Locking).

14 DevOps Pipelines

GitHub Actions for CI/CD. Multi-stage builds. Static analysis with Ruff, Mypy, and Bandit.

15 Technical Stack

FastAPI, ARQ, React, Postgres, Redis, Groq, Gemini, Sentry, Gunicorn.

16 Troubleshooting

Forensic lookup via correlation IDs. Readiness probes. Service connectivity diagnostics.
